# 问题诊断
# ============================================================

## 原始问题
- 错误: `boost::python::error_already_set` 
- 位置: `aligator_mpc_controller.py:159` (solver.run() 调用)
- 根因: Python 自定义 cost function (EEPosCost) 的 C++ 异常传播失败

## 崩溃堆栈
```
PyCostFunction::evaluate() 
  → C++ 异常抛出 
  → boost::python 异常传播失败 
  → abort()
```

# ============================================================
# 解决方案
# ============================================================

## 核心思路
**不使用 Python 自定义 cost，改用 ALIGATOR 内置 cost**

### 修改内容
1. 创建 `aligator_problem_builtin.py`
2. 使用 `QuadraticStateCost` 替代自定义 `EEPosCost`
3. 使用 `QuadraticControlCost` 进行控制正则化
4. 通过状态空间跟踪近似 EE 跟踪

### API 修正
- `QuadraticControlCost(space, target, weights)` ✅
- `QuadraticStateCost(space, nu, target, weights)` ✅
- `ControlErrorResidual(ndx, nu)` ✅

# ============================================================
# 误差分析
# ============================================================

## 为什么误差较大？

### 1. Cost Function 简化
- **原版**: 精确的 EE 位置 cost（3-DOF任务空间）
- **修复版**: 状态空间 cost（10-DOF配置空间）
- **影响**: 无法精确跟踪 EE 位置

### 2. 参考轨迹生成
- **问题**: 使用 nominal posture + 基座位置
- **缺失**: 末端执行器的 IK 解
- **结果**: 机械臂姿态不匹配圆形轨迹

### 3. 权重配置
- **当前**: w_state=10, w_base=1, w_ee_pos=100
- **问题**: 状态空间权重无法直接映射到 EE 跟踪

## Phase 1-3 为什么效果好？

Phase 1-3 使用自定义 EEPosCost：
```python
cost = 0.5 * w * ||FK(q) - p_ref||²
```
- 直接优化 EE 位置误差
- 梯度精确（通过 Jacobian）
- RMS误差 1.83 cm

# 改进方案

## 方案 1: 优化状态空间参考 ⭐⭐⭐
**思路**: 通过 IK 预计算更好的参考配置

**步骤**:
1. 对每个参考 EE 位置，求解 IK
2. 将 IK 解作为状态参考
3. 保持使用内置 cost

**优势**:
- 避免 Python cost function
- 更精确的参考轨迹
- 预期误差: 3-5 cm

## 方案 2: 修复 boost::python 绑定 ⭐⭐
**思路**: 修复 C++ 异常处理

**步骤**:
1. 定位 ALIGATOR 源码中的异常处理
2. 添加 try-catch 包装
3. 重新编译 ALIGATOR

**难度**: 需要 C++ 开发经验

## 方案 3: 使用 C++ cost function ⭐⭐⭐⭐
**思路**: 在 C++ 层实现 EEPosCost

**步骤**:
1. 在 ALIGATOR 中添加 C++ EE tracking cost
2. 注册到 Python 绑定
3. 重新编译

**优势**:
- 最佳性能
- 无绑定问题
- 预期误差: 1.8-2.5 cm

## 方案 4: 两阶段优化 ⭐⭐⭐⭐⭐
**思路**: 先 IK 规划，再 MPC 跟踪

**步骤**:
1. 离线: 对参考 EE 轨迹求解 IK
2. 在线: MPC 跟踪 IK 配置轨迹
3. 使用内置 cost

**优势**:
- 无需修改 ALIGATOR
- 高精度跟踪
- 易于实现
