## Phase 4 模型修复

### 问题诊断

**Phase 4核心问题**:
```
积分器不匹配:
  ALIGATOR: semi-implicit Euler (显式)
  MuJoCo:   implicitfast (隐式，多子步，数值阻尼)

结果:
  单步误差: 7e-5 (可接受)
  25步累积 (dt=0.05s): 0.034 (放大480倍!)
  
导致:
  MPC预测与实际执行严重偏差
  收敛率: 0%
  RMS误差: 2.5-5.0 cm
```

### 修复方案

**方案1: 减小MPC时间步长** (已实施) ✅

```python
# 修复前
dt = 0.05s
horizon = 10
预测视野 = 0.5s
累积误差 = 25步 × 7e-5 = 0.034

# 修复后
dt = 0.01s
horizon = 5
预测视野 = 0.05s
累积误差 = 5步 × 7e-5 = 0.007 (减少5倍)
```

**实施文件**: `scripts/test_phase4_fixed.py`

**预期效果**:
```
收敛率:   0% → 20-40%
RMS误差:  2.5-5.0cm → 1.5-3.0cm
累积误差: 减少5倍
代价:     MPC调用增加5倍
```

**方案2: 实现Implicit积分器** (Phase 8计划)

```
方法: 实现implicit Euler或RK4
效果: 彻底解决积分器不匹配
预期: <2.0cm误差, 80%+收敛率
工作量: 2-3周
```
