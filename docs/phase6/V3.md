# Phase 6-v3 使用指南

**推荐场景**: 双臂 UR5e，力矩 (motor) 执行器

---

## 两条路径

### Step 1 — 生产路径 (推荐)

```
IK 目标 (20 Hz) → 关节插值 (500 Hz) → τ_g(q_des) + PD → MuJoCo motor
```

```bash
export PYTHONPATH=.:../../build/bindings/python
pixi run -e all python scripts/test_phase6_v3_step1_simple.py
```

| 指标 (10 s 圆形) | 值 |
|------------------|-----|
| 平均 RMS | **2.28 cm** |
| IK 残差 | 0.004 cm |
| 力矩饱和率 | ~1% |

### Step 2 — 研究路径 (IK-informed 动力学 MPC)

```
IK x_ref 轨迹 → 动力学 MPC (h=5, fix_base) → 插值 → τ_mpc + PD → MuJoCo
```

```bash
pixi run -e all python scripts/test_phase6_v3_step2_simple.py
pixi run -e all python scripts/test_phase6_v3_benchmark.py   # 与 Step 1 对比
```

| 指标 | 值 |
|------|-----|
| 平均 RMS | 3.56 cm |
| MPC 严格收敛 | 0% |
| MPC 可用率 (EE<2 cm) | 100% |
| 求解时间 | ~46 ms |

## 关键文件

| 文件 | 作用 |
|------|------|
| `phase6_v3_common.py` | IK、插值、重力前馈、MPC 可用性判定 |
| `dual_arm_dynamics_mpc.py` | 动力学 MPC (`weight_preset="ik_informed"`) |
| `coordinate_mapping.py` | qpos ↔ ctrl 顺序映射 |
| `feedforward_pd_controller.py` | 力矩 PD |

模型:
- 力矩: `assets/wheeled_dual_ur5e_v2_torque.xml`
- Pinocchio: `assets/wheeled_dual_ur5e_v2.xml`

## 坐标顺序 (重要)

| 空间 | Base 顺序 |
|------|-----------|
| qpos / Pinocchio | `[x, y, yaw, z, left_arm×6, right_arm×6]` |
| MuJoCo ctrl | `[x, y, z, yaw, left_arm×6, right_arm×6]` |

输出力矩前必须调用 `q_to_ctrl()`。
