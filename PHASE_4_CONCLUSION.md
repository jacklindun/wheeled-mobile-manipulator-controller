## ⚠️ 已知问题

### 问题1：动力学模型与MuJoCo积分器不匹配

**现象**：
- 单步积分误差：7e-5（可接受）
- 25步累积误差：0.034（480倍放大！）
- 导致MPC预测与实际执行严重偏差

**根本原因**：
- ALIGATOR：semi-implicit Euler（显式，一步积分）
- MuJoCo：implicitfast（隐式，多子步，数值阻尼）
- 每步微小差异累积后变成巨大误差

**影响**：
- MPC优化出的控制在实际执行时完全不对
- 求解器无法收敛（收敛率0%）
- 权重调参无效

**诊断结果**：
```
测试：水平姿态机械臂，零扭矩
Pinocchio预测：shoulder_lift加速度 = -20.1 rad/s²（受重力下坠）
MuJoCo单步：   shoulder_lift加速度 = -0.0 rad/s²（静止）
差异原因：积分器的隐式处理和数值稳定化

单步对比（dt=0.002）：
  |Δq| = 7e-5  ✓ 可接受
  
25步累积（dt=0.05）：
  |Δq| = 0.034  ✗ 误差放大480倍
```

### 问题2：求解器不收敛

**现象**：
- 收敛率：0%（所有测试）
- 迭代数：50（达到上限）
- EE误差：2.5-5.0 cm（次优解）

**原因**：
- 动力学不匹配导致优化目标错误
- Warm-start基于错误的预测
- 权重调参无法改善（问题在模型层面）

**阶段1调参结果**：
```
配置           mu_init    收敛率    EE RMS
Baseline       1e-1       0.0%      2.88cm
mu_1e-2        1e-2       0.0%      2.88cm
mu_5e-2        5e-2       0.2%      2.88cm
mu_2e-2        2e-2       0.0%      2.88cm
```
→ 调参完全无效，证明问题不在参数层面

### 问题3：观察到的异常行为

**"红球后退"现象**：
- t=0s：红球（EE）和黄球（目标）几乎重合
- t=0-1s：红球突然向后移动，误差增大到5cm
- t=1-3s：红球重新追踪，误差降到2.5cm
- t>3s：稳定在2.5cm误差（次优解）

**原因**：
- t=0时从零初值开始优化
- 求解器不收敛，输出次优控制
- 次优控制在实际执行时因动力学不匹配产生错误运动
- 之后基于错误的warm-start继续优化，陷入次优循环

## 🛠️ 解决方案（Phase 8）

### 推荐方案：实现Implicit积分器

**技术路线**：
```python
# 在ALIGATOR中实现implicit Euler
# 求解非线性方程组：
# M(q_next) * (v_next - v) = dt * f(q_next, v_next, u)
# q_next = q + dt * v_next

# Newton迭代
for iter in range(max_newton_iters):
    residual = compute_residual(x_next, x, u, dt)
    jacobian = compute_jacobian(x_next, u, dt)
    delta = solve(jacobian, -residual)
    x_next += delta
    if norm(residual) < tol:
        break
```

**预期效果**：
- 单步误差：7e-5 → 1e-6
- 25步累积：0.034 → 0.001（减少34倍）
- 收敛率：0% → 80%+
- EE误差：2.5cm → <2.0cm
